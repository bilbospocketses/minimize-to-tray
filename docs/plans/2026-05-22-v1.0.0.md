# minimize-to-tray v1.0.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v1.0.0 of `minimize-to-tray` as a public OSS repo (`bilbospocketses/minimize-to-tray`) with a working compiled `.exe`.

**Architecture:** Single AutoHotkey v2 script, single process. Direct Win32 DllCalls for `Shell_NotifyIcon` / `ExtractIconEx` / `SetWinEventHook` to manage per-app-group tray icons alongside AHK's built-in always-visible script tray icon. Compiled to a self-contained `.exe` via Ahk2Exe.

**Tech Stack:** AutoHotkey v2.0, Ahk2Exe compiler, PowerShell (build script), ImageMagick (icon generation), git + gh CLI, MIT license.

**Spec:** [`docs/specs/2026-05-22-design.md`](../specs/2026-05-22-design.md).

**Testing approach:** Manual smoke walkthrough per spec — AHK has no standard test framework that justifies the setup cost for a single ~400-line script. The spec's 10-step smoke list is the acceptance test. Each task ends with a `Verify` step the implementer must run before moving on.

**Multi-session cwd discipline:** Every file path in this plan is absolute. Every `git` command uses `git -C "C:/Users/jscha/source/repos/minimize-to-tray"`. Do not `cd` into the repo.

**PowerShell compatibility:** All `.ps1` scripts shipped in the repo (currently just `build.ps1`) must be PowerShell 5.1 compatible — runs on stock Windows 10/11 without requiring pwsh 7+. The Claude Code PowerShell tool used during *plan execution* runs pwsh 7+, but that's just the verification harness — scripts that end up committed need to work for end users.

## Task 5: Shell_NotifyIcon DllCall wrappers

**Files:**
- Modify: `C:/Users/jscha/source/repos/minimize-to-tray/minimize-to-tray.ahk`

`NOTIFYICONDATAW` on x64 Windows is 976 bytes (the Unicode version with all current fields, including `hBalloonIcon`). Padding and offsets follow the Win32 SDK struct layout. AHK v2's `Buffer` + `NumPut`/`StrPut` handle this. `szTip` is at offset 40 and is 256 bytes (128 WCHARs).

- [ ] **Step 5.1: Add Shell_NotifyIcon wrapper functions**

Add a new section near the bottom of the script (before `OnTrayMessage`):

```ahk
;------------------------------------------------------------------------------
; Shell_NotifyIcon wrappers (NIM_ADD / NIM_MODIFY / NIM_DELETE)
;------------------------------------------------------------------------------
; NOTIFYICONDATAW size = 976 bytes on Windows 10/11 x64.
global NID_SIZE := 976

BuildNid(uid, flags, hIcon := 0, tooltip := "") {
    global NID_SIZE, scriptGuiHwnd, WM_TRAYCALLBACK
    nid := Buffer(NID_SIZE, 0)

    NumPut("UInt", NID_SIZE,         nid, 0)    ; cbSize
    NumPut("Ptr",  scriptGuiHwnd,    nid, 8)    ; hWnd
    NumPut("UInt", uid,              nid, 16)   ; uID
    NumPut("UInt", flags,            nid, 20)   ; uFlags
    NumPut("UInt", WM_TRAYCALLBACK,  nid, 24)   ; uCallbackMessage
    NumPut("Ptr",  hIcon,            nid, 32)   ; hIcon

    ; szTip is a WCHAR[128] at offset 40 (Unicode, 256 bytes).
    if (tooltip != "")
        StrPut(tooltip, nid.Ptr + 40, 127, "UTF-16")

    return nid
}

ShellNotifyAdd(uid, hIcon, tooltip) {
    global NIM_ADD, NIF_MESSAGE, NIF_ICON, NIF_TIP
    flags := NIF_MESSAGE | NIF_ICON | NIF_TIP
    nid := BuildNid(uid, flags, hIcon, tooltip)
    return DllCall("shell32\Shell_NotifyIconW", "UInt", NIM_ADD, "Ptr", nid.Ptr, "Int")
}

ShellNotifyModifyTooltip(uid, hIcon, tooltip) {
    global NIM_MODIFY, NIF_ICON, NIF_TIP
    flags := NIF_ICON | NIF_TIP
    nid := BuildNid(uid, flags, hIcon, tooltip)
    return DllCall("shell32\Shell_NotifyIconW", "UInt", NIM_MODIFY, "Ptr", nid.Ptr, "Int")
}

ShellNotifyDelete(uid) {
    global NIM_DELETE
    nid := BuildNid(uid, 0)
    return DllCall("shell32\Shell_NotifyIconW", "UInt", NIM_DELETE, "Ptr", nid.Ptr, "Int")
}
```

;------------------------------------------------------------------------------
; Helpers
;------------------------------------------------------------------------------
GetExeIcon(exePath) {
    ; Returns an HICON for the first large icon embedded in exePath,
    ; or 0 if extraction fails. Caller is responsible for DestroyIcon.
    hIconLarge := 0
    count := DllCall("shell32\ExtractIconExW"
                     , "WStr", exePath
                     , "Int",  0           ; nIconIndex (0 = first)
                     , "Ptr*", &hIconLarge
                     , "Ptr*", 0           ; phiconSmall (we don't want it)
                     , "UInt", 1
                     , "UInt")
    return (count >= 1 && hIconLarge) ? hIconLarge : 0
}

HwndToGroup(hwnd) {
    ; Reverse lookup: returns the processName of the group containing hwnd,
    ; or "" if not tracked.
    global Groups
    for procName, group in Groups {
        for trackedHwnd in group.windows {
            if (trackedHwnd == hwnd)
                return procName
        }
    }
    return ""
}
```

;------------------------------------------------------------------------------
; Core flow
;------------------------------------------------------------------------------
MinimizeFocused() {
    hwnd := WinGetID("A")
    if (!hwnd)
        return
    HideAndStash(hwnd)
}

HideAndStash(hwnd) {
    global Groups, nextTrayUid

    try {
        procName := ProcessGetName(WinGetPID("ahk_id " hwnd))
        exePath  := ProcessGetPath(WinGetPID("ahk_id " hwnd))
    } catch {
        return  ; window died or we can't see it
    }

    if (Groups.Has(procName)) {
        ; Push onto existing group
        group := Groups[procName]
        group.windows.Push(hwnd)
    } else {
        ; First window of this type — create new group + new tray icon
        hIcon := GetExeIcon(exePath)
        if (!hIcon) {
            ; Fall back to generic app icon
            hIcon := DllCall("LoadIconW", "Ptr", 0, "Ptr", 32512, "Ptr")
        }
        uid := nextTrayUid
        nextTrayUid++

        result := ShellNotifyAdd(uid, hIcon, procName)
        if (!result) {
            ; Retry once after 500ms
            Sleep(500)
            result := ShellNotifyAdd(uid, hIcon, procName)
        }
        if (!result) {
            MsgBox("Shell_NotifyIcon(NIM_ADD) failed twice for " procName ". Aborting minimize.",
                   "minimize-to-tray", "IconX")
            DllCall("DestroyIcon", "Ptr", hIcon)
            return
        }

        Groups[procName] := { windows: [hwnd], trayUid: uid, hIcon: hIcon }
        group := Groups[procName]
    }

    WinHide("ahk_id " hwnd)

    ; Update tooltip with count
    tooltip := procName " (" group.windows.Length " window" (group.windows.Length == 1 ? "" : "s") ")"
    ShellNotifyModifyTooltip(group.trayUid, group.hIcon, tooltip)
}
```

- [ ] **Step 8.1: Replace the stub `OnTrayMessage` with real dispatch**

Replace the existing `OnTrayMessage` body with:

```ahk
OnTrayMessage(wParam, lParam, msg, hwnd) {
    global Groups, WM_LBUTTONUP

    uid := wParam
    event := lParam & 0xFFFF  ; low word is the mouse event

    ; Find the group with this trayUid
    foundProcName := ""
    for procName, group in Groups {
        if (group.trayUid == uid) {
            foundProcName := procName
            break
        }
    }
    if (foundProcName == "")
        return

    group := Groups[foundProcName]

    if (event == WM_LBUTTONUP) {
        ; Pop the top of the stack and restore
        if (group.windows.Length == 0) {
            DestroyGroup(foundProcName)
            return
        }
        targetHwnd := group.windows.Pop()
        try {
            WinShow("ahk_id " targetHwnd)
            WinActivate("ahk_id " targetHwnd)
        }
        ; If stack now empty, destroy group; else update tooltip
        if (group.windows.Length == 0) {
            DestroyGroup(foundProcName)
        } else {
            tooltip := foundProcName " (" group.windows.Length " window" (group.windows.Length == 1 ? "" : "s") ")"
            ShellNotifyModifyTooltip(group.trayUid, group.hIcon, tooltip)
        }
    }
    ; Right-click handling added in Task 10
}
```

    global Groups
    if (!Groups.Has(procName))
        return

    ; Remove hwnd from the stack
    for i, h in group.windows {

            group.windows.RemoveAt(i)
            break
        }

    global Groups
    if (!Groups.Has(procName))
        return
    group := Groups[procName]
    ; Restore top-of-stack first
    i := group.windows.Length
    while (i >= 1) {
        hwnd := group.windows[i]
        try {
            WinShow("ahk_id " hwnd)
            WinActivate("ahk_id " hwnd)
        }
        i--
    }
    group.windows := []
    DestroyGroup(procName)
}

    global Groups
    if (!Groups.Has(procName))
        return
    group := Groups[procName]
    ; Clone the array because WinClose may trigger the event hook which mutates the stack
    snapshot := group.windows.Clone()
    for hwnd in snapshot {
        try WinClose("ahk_id " hwnd)
    }
    ; The EVENT_OBJECT_DESTROY hook (Task 12) handles bookkeeping as windows die.

    MouseGetPos(&x, &y, &hwnd)
    if (!hwnd)
        return false

    ; AHK's SendMessage returns the result.
    lParam := ((y & 0xFFFF) << 16) | (x & 0xFFFF)

    if (rootHwnd)

## Why

Windows doesn't ship with minimize-to-tray. The existing utilities in this space (RBTray, MinTrayR, TrayIt, various AutoHotkey scripts) are stale, fork-fragmented, or behave inconsistently on modern Windows. This is a focused v1 with two triggers, zero configuration, and consistent behavior across modern Win32 apps.

First-run note: Windows SmartScreen will warn about the unsigned executable. Click **More info** → **Run anyway**. Code signing is a planned follow-up.

## Usage

Two ways to minimize the current window to the system tray:

- **`Win+Shift+Z`** — minimize the focused window.
- **Middle-click on a window's title bar** — minimize that window.

Windows of the same app collapse under a single tray icon (showing the app's own icon):

- **Left-click** a tray icon → restore the most recently minimized window (LIFO).
- **Right-click** a tray icon → menu lists every minimized window of that app, plus *Restore All* and *Close All*.

The app's own always-visible tray icon (separate from the per-app icons) shows the shortcuts on hover, and right-clicks to **About** / **Exit**.

## Known limitations (v1.0.0)

- **Custom title bars**: Chrome, Edge, Office, VS Code, and other apps that draw their own title bars only register middle-click on the drag-handle / caption region. Middle-clicking on a Chrome tab, for example, will not trigger minimize — the tab consumes the click.

- **SmartScreen warning** on first run for the unsigned .exe.
- **Windows 11 collapsed-tray-icons mode**: the per-app tray icons may hide in the overflow flyout — functions identically, but you'll want to pin them if you prefer them always-visible.
